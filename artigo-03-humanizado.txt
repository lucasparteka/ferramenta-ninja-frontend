Para criar um QR Code PIX, você precisa ter uma chave PIX cadastrada em alguma instituição financeira. A chave é o identificador que vincula o QR Code ao seu CPF ou CNPJ no sistema do Banco Central.

Muda se você trocar de número. — E-mail: útil para quem prefere não compartilhar CPF ou telefone. — Chave aleatória (EVP): uma sequência de letras e números gerada automaticamente. Para gerar um QR Code, você vai precisar de pelo menos uma chave cadastrada e ativa. Mas isso pode depender do caso.

[QR Code estático x QR Code dinâmico]

Essa é a diferença mais importante de entender antes de gerar o código. Não exatamente — e aí?

O QR estático é uma imagem fixa gerada a partir da sua chave Pix. Você pode: — Deixar o valor em aberto (o pagador digita o valor na hora) — Pré-definir um valor fixo (útil para produtos com preço fixo)

Use quando o atendimento for presencial, a cobrança for informal ou envolver pessoas físicas e MEIs que não precisam de conciliação automática. É o tipo mais simples e funciona bem para a maioria dos casos do dia a dia.

Limitação: como o QR code é sempre o mesmo, você não consegue identificar qual pagamento corresponde a qual venda — a menos que faça esse controle manualmente. Se dez clientes comprarem ao mesmo tempo, todos pagarão para o mesmo QR code. Não é tão simples assim.

O QR dinâmico é gerado por transação — um código diferente para cada cobrança. Ele pode conter: — Valor exato da cobrança — Data de vencimento — Nome do pagador — Informações para conciliação automática

Quando usar: e-commerce, sistemas de gestão, cobranças com vencimento e negócios que precisam rastrear cada pagamento automaticamente.

Limitação: para gerar QR dinâmico, você precisa de acesso à API PIX de alguma instituição financeira ou de um sistema de cobrança integrado (como as plataformas de pagamento). Não dá para gerar manualmente.

Para a maioria dos autônomos e pequenos negócios, o QR estático resolve bem — e faz diferença.

A principal diferença entre **QR Code PIX** e **PIX copia e cola** está na forma como os dados da transação são compartilhados entre pagador e recebedor: ### **QR Code PIX** — É um **código visual** (geralmente uma imagem quadrada com padrões pretos e brancos) que contém todas as informações necessárias para realizar um pagamento PIX. — Pode ser **estático** (mesmo QR Code para várias cobranças, sem valor fixo ou com valor fixo) ou **dinâmico** (gerado por cobrança única, com valor e outras informações específicas). — O pagador **usa a câmera do celular** (via app do banco ou carteira digital) para **ler o QR Code** e confirmar o pagamento. — Muito usado em **lojas físicas**, **sites**, **redes sociais** ou **impressos**. ### **PIX Copia e Cola** — É uma **string de texto alfanumérica longa** (começa com `pix. ... `) que também contém todas as informações da transação. — O recebedor **copia esse código** e o envia ao pagador (por mensagem, e-mail, WhatsApp etc. ). — O pagador **cola o código no campo específico do app do banco** (normalmente chamado de “Colar” ou “PIX Copia e Cola”) para carregar os dados da transação e confirmar o pagamento. — Muito útil quando **não é possível exibir ou escanear um QR Code**, como em conversas por texto. ### Em resumo: — **QR Code PIX**: escaneado visualmente com a câmera. — **PIX Copia e Cola**: colado como texto no app bancário. Ambos levam à mesma funcionalidade — realizar um pagamento instantâneo via PIX — , mas usam meios diferentes para transmitir os dados da transação.

São representações diferentes do mesmo payload. O QR Code é a versão visual, feita para escanear com a câmera. Já o Pix copia e cola é a versão em texto — aquela sequência longa de caracteres que você cola no app do banco. Ambos contêm as mesmas informações e podem ser gerados juntos.
